# Порівняння алгоритмів видачі решти

## Опис завдання

Метою було реалізувати два алгоритми для видачі решти монетами `[50, 25, 10, 5, 2, 1]`:

1. **Жадібний алгоритм** (`find_coins_greedy`): Завжди вибирає найбільшу доступну монету.
2. **Динамічне програмування** (`find_min_coins`): Знаходить гарантовано мінімальну кількість монет для будь-якої системи номіналів.

## Результати вимірювань

Час виконання в секундах (приблизні значення):

| Сума (Amount) | Greedy Time (s) | DP Time (s) |
| ------------- | --------------- | ----------- |
| 113           | 0.000001        | 0.000020    |
| 1000          | 0.000001        | 0.000178    |
| 2000          | 0.000001        | 0.000317    |
| 5000          | 0.000000        | 0.000741    |

## Аналіз ефективності

### Жадібний алгоритм

- **Часова складність**: $O(K)$, де $K$ - кількість номіналів монет. Завдяки оптимізації (використання ділення замість віднімання), час виконання практично миттєвий і не залежить від суми решти.
- **Переваги**: Дуже швидкий, використовує мінімум пам'яті $O(1)$.
- **Недоліки**: Для деяких наборів монет може не дати оптимального рішення. Наприклад, для монет `[9, 6, 1]` і суми 12, жадібний дасть `9+1+1+1` (4 монети), а оптимально `6+6` (2 монети).

### Динамічне програмування

- **Часова складність**: $O(A \cdot K)$, де $A$ - сума решти, а $K$ - кількість номіналів. Час зростає лінійно зі збільшенням суми.
- **Просторова складність**: $O(A)$ для збереження таблиці мінімальних кількостей та відновлення шляху.
- **Переваги**: Гарантує знаходження мінімальної кількості монет для **будь-якого** набору монет.
- **Недоліки**: Значно повільніший за жадібний алгоритм на великих сумах. Вимагає багато пам'яті для великих сум.

## Висновки

Для канонічних монетних систем (як у цьому завданні: 50, 25, 10, 5, 2, 1) жадібний алгоритм є **безумовним переможцем**. Він дає той самий оптимальний результат, що й алгоритм динамічного програмування, але працює на порядки швидше і не вимагає додаткової пам'яті.

Динамічне програмування варто використовувати лише у випадках, коли набір монет є специфічним і не дозволяє застосовувати жадібний підхід для отримання оптимального результаку, або коли розміри суми невеликі, і точність є критичнішою за швидкість
